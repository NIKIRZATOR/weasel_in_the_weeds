# Каталог `assets`

Здесь лежат графические ресурсы игры.

## Основные подпапки

- `characters/` — спрайты персонажей, NPC и связанные изображения.
- `tiles/` — тайлы земли, препятствий и декоративных объектов.
- `items/` — иконки предметов для инвентаря, хотбара и крафта.

## Структура `items/`

Рекомендуемая структура такая:

- `items/materials/`
- `items/consumables/`
- `items/weapons/`
- `items/accessories/`
- `items/quest/`
- `items/currency/`

## Как добавлять иконку предмета

1. Положить PNG-файл в подходящую подпапку `assets/items/...`
2. Указать путь в `game/items/catalog_data.json` через поле `icon_path`

Пример:

```json
"grass_blade": {
  "name": "Grass Blade",
  "kind": "material",
  "stackable": true,
  "max_stack": 999999,
  "icon_path": "items/materials/grass_blade.png",
  "description": "A fresh blade of grass."
}
```

## Важные правила

- использовать `png` с прозрачностью;
- держать один стиль иконок;
- желательно одинаковый размер исходников, например `32x32` или `48x48`;
- путь в `icon_path` указывается относительно каталога `assets/`.

## Что будет, если иконки нет

Если файл не найден или `icon_path` не задан, интерфейс рисует текстовую заглушку из двух букв. Это позволяет добавлять иконки постепенно, не ломая UI.
