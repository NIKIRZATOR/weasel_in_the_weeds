# Редактор предметов и рецептов

Инструмент находится в файле:

```text
dev_tools/item_recipe_editor/item_recipe_editor.py
```

Запускать из корня проекта:

```bash
python dev_tools/item_recipe_editor/item_recipe_editor.py
```

## Что умеет

Редактор работает напрямую с файлами:

- `game/items/catalog_data.json`
- `game/crafting/recipes_data.json`

Он позволяет:

- смотреть список уже созданных предметов;
- создавать новые предметы;
- редактировать существующие предметы;
- дублировать и удалять предметы;
- смотреть список уже созданных рецептов;
- создавать новые рецепты;
- редактировать существующие рецепты;
- дублировать и удалять рецепты;
- сохранять изменения обратно в те же JSON-файлы.

## Вкладка "Предметы"

Можно редактировать:

- `id`
- `name`
- `kind`
- `stackable`
- `max_stack`
- `price`
- `icon_path`
- `wallet_key`
- `equip_slot`
- `inventory_capacity_bonus`
- `description`
- `stats`

### Как задавать иконку

Поле `icon_path` указывается относительно каталога `assets/`.

Пример:

```text
items/materials/grass_blade.png
```

### Как работает `wallet_key`

Если предмет имеет тип `currency`, можно указать `wallet_key`, чтобы он не попадал в инвентарь, а сразу уходил в отдельный счетчик ресурса.

Примеры:

- `coins`
- `knowledge_shards`

## Вкладка "Рецепты"

Можно редактировать:

- `id`
- `name`
- `category`
- `description`
- `result`
- `unlock_type`
- `knowledge_cost`
- `sort_order`
- `required_flags`
- `ingredients`

### Формат ингредиентов

Каждый ингредиент пишется с новой строки:

```text
stick:1
stone_pebble:2
```

То есть:

- слева `item_id`
- справа количество

## Важное поведение

- список доступных предметов для результата рецепта берется из текущего каталога предметов;
- если переименовать `item_id`, редактор обновит ссылки на этот предмет в рецептах;
- если у предмета нет иконки или файл не найден, игра покажет текстовую заглушку.
