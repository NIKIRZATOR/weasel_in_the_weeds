# Инструменты проекта

## Редактор диалогов

Редактор диалогов находится в файле:

```text
tools/dialogue_editor.py
```

Запускать из корня проекта:

```bash
python tools/dialogue_editor.py
```

Редактор создает и изменяет JSON-файлы диалогов в папке:

```text
dialogues/
```

Такие файлы потом можно подключать к NPC в уровне через `dialogue_file`:

```json
{
  "type": "npc_object",
  "name": "Hermit Mouse",
  "x": 5,
  "y": 3,
  "width": 1,
  "height": 1,
  "properties": {
    "health": 35,
    "dialogue_file": "hermit_mouse.json"
  }
}
```

## Что можно редактировать

В редакторе можно настроить:

- id диалога;
- стартовый узел;
- id каждого узла;
- говорящего: `npc` или `player`;
- текст реплики;
- переход к следующему узлу через `next`;
- варианты ответа игрока;
- награды за попадание в узел;
- монеты;
- предметы;
- сюжетные флаги.

## Как устроен диалог

Диалог состоит из узлов. Узел - это одна реплика или один экран диалога.

У каждого узла есть:

- `id` - уникальное имя узла;
- `speaker` - кто говорит;
- `text` - текст реплики;
- `next` - следующий узел, если диалог линейный;
- `choices` - варианты ответа, если игрок должен выбрать реплику;
- `rewards` - награды, которые выдаются при входе в этот узел.

Если у узла есть варианты ответа, то `next` у самого узла не используется. Переход задается у каждого варианта отдельно.

## Основной порядок работы

1. Нажать `New`, если нужен новый диалог.
2. Указать `Dialogue id`, например `hermit_mouse`.
3. Создать нужные узлы кнопкой `Add`.
4. Для каждого узла указать `Id`, `Speaker`, текст и переходы.
5. Выбрать стартовый узел в поле `Start`.
6. Нажать `Validate`, чтобы проверить битые ссылки.
7. Нажать `Save` или `Save As`.

## Варианты ответа

В блоке `Choices` можно добавить несколько вариантов.

Для каждого варианта указывается:

- текст варианта;
- следующий узел, куда ведет выбор.

В игре выбор работает так:

- `W` / `Up` - выбрать вариант выше;
- `S` / `Down` - выбрать вариант ниже;
- `1-9` - выбрать вариант по номеру;
- `Enter`, `Space`, `E` - подтвердить выбранный вариант;
- `Esc` - закрыть диалог.

## Награды

Награды задаются в блоке `Rewards`.

Монеты:

```text
3
```

Предметы пишутся по одному на строку:

```text
map
stick:2
small_backpack
```

Формат предмета:

- `item_id` - выдать 1 предмет;
- `item_id:quantity` - выдать указанное количество.

Предмет должен существовать в:

```text
game/items/catalog.py
```

Флаги пишутся по одному на строку:

```text
met_mouse_hermit
has_map
```

Флаги сохраняются у игрока в `Player.story_flags`.

Награда выдается один раз для конкретного NPC и конкретного узла. Это нужно, чтобы игрок не мог бесконечно получать одни и те же предметы, повторяя диалог.

## Проверка диалога

Кнопка `Validate` проверяет:

- указан ли стартовый узел;
- существует ли стартовый узел;
- ведут ли `next` и варианты ответа к существующим узлам.

Если есть ошибки, редактор покажет предупреждение.

## Уже готовый пример

Пример диалога:

```text
dialogues/hermit_mouse.json
```

Он подключен к NPC `Hermit Mouse` во втором уровне:

```text
levels/level_02.json
```

Диалог содержит 7 узлов и выдает:

- `small_backpack`;
- `map`;
- 3 монеты;
- флаги `met_mouse_hermit` и `has_map`.
