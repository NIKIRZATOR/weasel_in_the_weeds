# Boss Ideas

Этот файл нужен как рабочая заметка по боссам для проекта.

Здесь собраны:

- подход к дизайну боссов;
- подходящие типы существ под сеттинг;
- идеи по фазам, атакам и аренам;
- кандидат на первого босса;
- технические заметки по реализации.

## Общий подход

Босс в этой игре должен быть не просто "большим врагом с большим HP", а отдельной encounter-системой.

Хороший босс здесь должен опираться на 4 вещи:

- образ существа;
- собственный набор атак;
- механику арены;
- фазность.

Босс должен ощущаться как:

- магическое животное;
- дух леса, болота, норы или руин;
- важный хранитель территории;
- испытание на понимание боевой системы.

## Что должен уметь хороший босс

Минимально у босса должны быть:

- базовые атаки;
- сигнатурные атаки;
- 2 фазы;
- recovery-окна;
- telegraph/windup;
- взаимодействие с ареной.

Хорошие механики для боссов:

- `windup + dodge window`;
- `area denial`;
- `summons`;
- `phase transition`;
- `vulnerability window`;
- `positioning check`;
- `anti-greed punish`.

## Подходящие боссы по сеттингу

### 1. Forest Guardian

Образ:

- большой олень;
- рогатый дух леса;
- магический хранитель чащи.

Темы:

- корни;
- таран;
- шипы;
- контроль арены;
- дух природы.

Почему хороший первый босс:

- хорошо ложится на лесной сеттинг;
- читается визуально;
- не требует сложной магической логики на первом шаге;
- хорошо обучает позиционированию и dodging.

### 2. Spider Matron

Образ:

- огромная лесная паучиха;
- магическое паучье чудовище;
- хранительница паутины и коконов.

Темы:

- web trap;
- poison;
- egg summon;
- slow zones;
- add management.

### 3. Crow Shaman

Образ:

- гигантский ворон;
- дух-колдун;
- лесной предвестник.

Темы:

- ranged magic;
- curse zones;
- feather projectiles;
- blink / reposition;
- support summons.

### 4. Bear Spirit

Образ:

- древний медвежий дух;
- тяжелый guardian boss.

Темы:

- мощные melee удары;
- roar;
- leap slam;
- fury phase;
- short-range dominance.

### 5. Fox Queen

Образ:

- мистическая лисица;
- обманщица;
- быстрая босс-сущность.

Темы:

- illusions;
- dash;
- fake openings;
- burning trails;
- mobility duel.

### 6. Toad Archmage

Образ:

- странное болотное магическое существо;
- тяжелый кастер-зверь.

Темы:

- poison puddles;
- bubble volleys;
- tongue lash;
- summon slimes / spores;
- zone denial.

### 7. Old Owl

Образ:

- древняя сова-хранитель;
- ночной дух.

Темы:

- gaze beams;
- radial waves;
- darkness;
- delayed attacks;
- arena awareness.

## Кандидат на первого босса

Лучший кандидат для первой реализации:

- `ForestGuardianBoss`

Причины:

- он хорошо соответствует миру;
- его проще реализовать на текущем коде;
- он естественно использует melee, charge, roots и simple area control;
- на нем можно обкатать фазовую систему и boss state machine.

## ForestGuardianBoss

### Образ

- большой олень-дух;
- рогатый лесной хранитель;
- древнее магическое существо, защищающее проход или священную поляну.

### Боевая роль

- pressure boss;
- area control boss;
- short-to-mid range threat;
- phase escalator.

### Фаза 1

Задача фазы:

- научить игрока читать windup;
- уклоняться от прямолинейных атак;
- не стоять слишком близко слишком долго.

Атаки:

1. `Horn Thrust`
- быстрый укол вперед;
- line / thrust hitbox;
- небольшое recovery.

2. `Root Sweep`
- дуговой удар вокруг передней полусферы;
- arc hitbox;
- средний windup;
- хороший telegraph.

3. `Short Charge`
- короткий таран по прямой;
- сильный урон, если игрок стоит на линии;
- после промаха дает окно для контратаки.

### Фаза 2

Активируется, например, на `60%` или `50% HP`.

Что меняется:

- атаки становятся быстрее;
- появляется контроль арены;
- добавляется summons/hazards.

Новые механики:

1. `Root Burst`
- из земли вырастают корни в нескольких точках;
- delayed area attack;
- заставляет двигаться.

2. `Summon Sprouts`
- призыв 2 небольших существ или корневых узлов;
- adds не должны быть слишком сильными;
- их роль — мешать игроку во время reposition.

3. `Enraged Charge`
- более длинный и опасный разгон;
- сильнее punish за плохую позицию.

### Арена

Подходит:

- круглая поляна;
- алтарь природы;
- лесной clearing;
- ограниченное пространство с несколькими безопасными зонами.

Арена может использовать:

- корни как временные препятствия;
- кусты/камни как якоря позиционирования;
- зоны, где появляются шипы.

### Окна уязвимости

Очень важно дать игроку честные моменты для ответа.

Примеры:

- после `Short Charge`, если босс не попал;
- после большого `Root Sweep`;
- после завершения `Root Burst`.

Именно в эти окна игрок должен иметь шанс:

- дать heavy;
- дать charged;
- отстрелять из лука;
- восстановить темп.

## Технический подход к реализации

Босс не должен строиться как "просто MeleeEnemy, но жирнее".

Лучший путь:

- `BaseBossEnemy` как отдельная база;
- phase thresholds;
- boss action selection;
- attack queue / cooldown logic;
- arena hooks;
- support для telegraph и vulnerability windows.

### Что должен хранить BaseBossEnemy

Минимально:

- текущая фаза;
- список фазовых порогов;
- текущая активная атака;
- boss-only cooldowns;
- специальные таймеры;
- флаг перехода между фазами;
- support под summon/hazard spawning.

### Что должен уметь конкретный босс

Конкретный босс должен определять:

- список атак;
- условия перехода фазы;
- выбор следующей атаки;
- boss arena interactions;
- summon behavior;
- уникальные hitbox-профили атак.

## Пример награды за босса

После победы над боссом можно давать:

- гарантированный XP burst;
- флаг сюжета;
- новое оружие;
- редкий crafting material;
- ключ / пропуск / сюжетный артефакт;
- checkpoint unlock;
- открытие следующей локации.

Для `ForestGuardianBoss` подойдут:

- рог / essence / spirit bark;
- уникальный материал на копье/лук;
- доступ к следующей зоне.

## Чего не делать

Не стоит:

- делать босса только "толстым";
- делать 5 фаз сразу;
- забивать арену постоянным мусором;
- давать атаки без telegraph;
- делать суммон бесконечным и хаотичным;
- превращать бой в DPS-check без позиционирования.

## Рекомендуемый порядок реализации первого босса

1. Сделать `BaseBossEnemy`.
2. Ввести 2 фазы.
3. Реализовать 3 базовые атаки.
4. Добавить 1 mechanic арены.
5. Добавить 1 summon или root hazard.
6. Дать vulnerability windows.
7. Потом уже делать boss HUD / intro / victory sequence.

## Краткий вывод

Если нужен первый практичный босс для проекта, самый сильный стартовый выбор:

- `ForestGuardianBoss`

Он:

- соответствует миру;
- не слишком сложен для первой реализации;
- хорошо использует текущую боевую систему;
- дает хороший фундамент для всех следующих боссов.
