# Objects

Папка `game/objects` содержит объекты мира, которые загружаются из JSON-файлов уровней.

Объекты из этой папки отличаются от сущностей (`game/entities`): они обычно являются частью карты. Например: камень, сундук, предмет на земле, зона перехода между уровнями. Враги и игрок находятся в `game/entities`, даже если они тоже описаны в JSON уровня.

## Как объект попадает в игру

1. Уровень загружается через `game/world/level.py`.
2. `GameScene._load_world_objects()` перебирает список `objects` из JSON.
3. Для обычных объектов вызывается `create_world_object()` из `factory.py`.
4. Фабрика смотрит на поле `type` и создает нужный класс.

Координаты и размеры в JSON задаются в тайлах, а фабрика переводит их в пиксели:

```json
{
  "type": "solid_object",
  "name": "Stone Block",
  "x": 4,
  "y": 6,
  "width": 1,
  "height": 1
}
```

Если размер не указан, используется `width: 1` и `height: 1`.

## Общий формат JSON-объекта

```json
{
  "type": "object_type",
  "name": "Object Name",
  "x": 0,
  "y": 0,
  "width": 1,
  "height": 1,
  "solid": false,
  "properties": {}
}
```

Поля:

- `type` - тип объекта. По нему фабрика выбирает класс.
- `name` - имя объекта. Может использоваться в сообщениях или как `item_id` для подбираемых предметов.
- `x`, `y` - позиция объекта на карте в тайлах.
- `width`, `height` - размер объекта в тайлах.
- `solid` - используется для `interactable_object`, если он должен блокировать движение.
- `properties` - дополнительные настройки конкретного объекта.

## WorldObject

Файл: `world_object.py`

Базовый класс для всех объектов мира.

Хранит:

- позицию и размер;
- хитбокс и зону взаимодействия через базовый `Entity`;
- `name`;
- `color`;
- `is_solid`;
- `is_interactable`;
- `properties`;
- `is_active`.

Метод `interact(player, game_scene)` по умолчанию ничего не делает и возвращает `False`.

Метод `draw(screen, camera)` рисует объект простым прямоугольником. Наследники могут переопределять отрисовку.

## SolidObject

Файл: `solid_object.py`

Непроходимый объект. Используется для камней, стен, завалов, стволов деревьев и других препятствий.

JSON:

```json
{
  "type": "solid_object",
  "name": "Stone Block",
  "x": 4,
  "y": 6,
  "width": 1,
  "height": 1
}
```

Особенности:

- `is_solid = True`;
- блокирует движение через `CollisionSystem`;
- не является интерактивным.

## InteractableObject

Файл: `interactable_object.py`

Базовый интерактивный объект. Сейчас работает как простой переключатель: при взаимодействии меняет активное состояние и цвет.

JSON:

```json
{
  "type": "interactable_object",
  "name": "Training Chest",
  "x": 6,
  "y": 6,
  "width": 1,
  "height": 1,
  "solid": false
}
```

Особенности:

- `is_interactable = True`;
- игрок может взаимодействовать с объектом, когда его зона взаимодействия пересекается с зоной игрока;
- сейчас взаимодействие вызывается клавишей `E` через `GameScene.try_interact()`;
- поле `solid` задается прямо в JSON, не внутри `properties`.

Позже этот класс можно расширить или заменить наследниками: сундук, дверь, рычаг, NPC-объект.

## PickableObject

Файл: `pickable_object.py`

Предмет, который можно поднять с земли.

JSON:

```json
{
  "type": "pickable_object",
  "name": "stick",
  "x": 5,
  "y": 17,
  "width": 1,
  "height": 1
}
```

Можно явно указать предмет и количество:

```json
{
  "type": "pickable_object",
  "name": "Bundle",
  "x": 5,
  "y": 17,
  "properties": {
    "item_id": "stick",
    "quantity": 3
  }
}
```

Можно выдать монеты:

```json
{
  "type": "pickable_object",
  "name": "Coins",
  "x": 5,
  "y": 7,
  "properties": {
    "coins": 5
  }
}
```

Особенности:

- `is_interactable = True`;
- после успешного подбора объект удаляется из `world_objects`;
- если `properties.item_id` не указан, используется `name`;
- предмет должен существовать в `game/items/catalog.py`;
- валюта из `ItemKind.CURRENCY` автоматически добавляется в монеты игрока.

## LevelTransition

Файл: `level_transition.py`

Зона перехода между уровнями. Когда хитбокс игрока входит в зону, игра проверяет условия. Если условия выполнены, экран темнеет, запускается загрузочный экран, затем создается новая `GameScene`.

Текущий `Player` передается в новую сцену, поэтому сохраняются:

- здоровье;
- выносливость;
- монеты;
- инвентарь;
- экипировка;
- хотбар;
- сюжетные флаги.

Минимальный JSON:

```json
{
  "type": "level_transition",
  "name": "To Forest Edge",
  "x": 18,
  "y": 27,
  "width": 1,
  "height": 3,
  "properties": {
    "target_level": "level_02.json",
    "target_spawn": {
      "x": 2,
      "y": 2
    }
  }
}
```

Полный пример с условиями:

```json
{
  "type": "level_transition",
  "name": "To Deep Forest",
  "x": 18,
  "y": 10,
  "width": 1,
  "height": 3,
  "properties": {
    "target_level": "level_03.json",
    "target_spawn": {
      "x": 2,
      "y": 2
    },
    "required_items": [
      "map",
      {
        "item_id": "stick",
        "quantity": 2
      }
    ],
    "required_coins": 10,
    "required_flags": ["met_mouse_hermit"],
    "set_flags": ["entered_deep_forest"],
    "blocked_message": "Проход пока закрыт."
  }
}
```

Поддерживаемые `properties`:

- `target_level` - имя JSON-файла уровня в папке `levels`.
- `target_spawn` - точка появления на следующем уровне в тайлах.
- `required_items` - список обязательных предметов. Можно указать строкой или объектом с `item_id` и `quantity`.
- `required_coins` - минимальное количество монет у игрока.
- `required_flags` - список сюжетных флагов, которые должны быть у игрока.
- `set_flags` - список флагов, которые будут выставлены при успешном переходе.
- `blocked_message` - сообщение, если условия не выполнены.

Флаги хранятся в `Player.story_flags`.

Примеры флагов:

- `met_mouse_hermit`;
- `has_map`;
- `boss_defeated`;
- `entered_forest_edge`;
- `entered_deep_forest`.

## player_spawn

`player_spawn` может быть указан в списке объектов уровня, но фабрика его игнорирует и возвращает `None`.

Основная точка появления игрока берется из верхнего поля уровня:

```json
{
  "player_spawn": {
    "x": 2,
    "y": 2
  }
}
```

Для переходов между уровнями используется `properties.target_spawn`.

## Как добавить новый тип объекта

1. Создать новый файл в `game/objects`, например `door_object.py`.
2. Сделать класс-наследник от `WorldObject` или одного из существующих объектов.
3. Добавить импорт и ветку создания в `factory.py`.
4. При необходимости добавить класс в `__init__.py`.
5. Описать JSON-формат в этом README.
6. Добавить объект в `levels/*.json` и проверить загрузку уровня.

Пример ветки в `factory.py`:

```python
if object_type == "door_object":
    return DoorObject(
        x,
        y,
        width,
        height,
        name=name,
        properties=properties,
    )
```

## Текущие ограничения

- Отрисовка объектов пока в основном простыми прямоугольниками.
- `InteractableObject` пока не различает действия вроде открыть, закрыть, толкать, тянуть.
- Подбор и взаимодействие сейчас используют одну кнопку в `GameScene`.
- Условия переходов проверяются только при входе игрока в зону.
- Нет сохранения прогресса на диск, поэтому флаги живут только в текущей игровой сессии.
