# Каталог `game/crafting`

Этот каталог отвечает за рецепты и правила крафта.

## Файлы

- `recipes_data.json` — библиотека рецептов.
- `models.py` — структуры данных рецепта:
  - ингредиенты,
  - результат,
  - описание рецепта.
- `catalog.py` — загрузка JSON и преобразование в объекты Python.
- `__init__.py` — удобный экспорт для импорта из других частей игры.

## Как работает крафт

1. Сцена крафта запрашивает список рецептов через `get_recipe_definitions()`.
2. Игрок выбирает рецепт.
3. Система проверяет:
   - открыт ли рецепт;
   - хватает ли `knowledge_shards`, если рецепт открывается за знания;
   - хватает ли материалов;
   - есть ли место в инвентаре для результата.
4. Если рецепт еще закрыт, игрок может один раз открыть его за осколки знаний.
5. После открытия крафт предмета тратит только обычные материалы.

## Типы открытия рецептов

- `default` — открыт сразу.
- `knowledge` — открывается за `knowledge_shards`.
- при необходимости можно добавить и другие типы, например сюжетные или через NPC.

## Где хранится прогресс

Прогресс открытия рецептов и текущие `knowledge_shards` хранятся у игрока в `game/entities/player.py`.
