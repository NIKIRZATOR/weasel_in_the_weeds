# Каталог `game/items`

Этот каталог отвечает за предметы и все, что связано с их хранением и отображением.

## Файлы

- `catalog_data.json` — библиотека предметов.
- `catalog.py` — загрузка предметов из JSON и загрузка иконок с кешем.
- `models.py` — описания предметов и стеков.
- `inventory.py` — обычный контейнер ячеек.
- `equipment.py` — экипированные предметы и бонусы.
- `types.py` — типы предметов и слоты экипировки.
- `__init__.py` — общий экспорт.

## Как это работает

1. Все предметы описываются в `catalog_data.json`.
2. При подборе, награде или крафте создается `ItemStack`.
3. Игрок складывает предмет:
   - в обычный инвентарь,
   - в сюжетный инвентарь,
   - в хотбар,
   - или в экипировку.

## Поля предмета

Основные поля:

- `name`
- `kind`
- `stackable`
- `max_stack`
- `price`
- `description`

Дополнительные поля:

- `equip_slot` — для экипируемых предметов;
- `stats` — бонусы к характеристикам;
- `inventory_capacity_bonus` — бонус к вместимости;
- `icon_path` — путь к иконке предмета относительно `assets/`.
- `wallet_key` — имя отдельного счетчика ресурса, если предмет является валютой и не должен храниться в инвентаре.

## Иконки предметов

Иконка задается в `catalog_data.json`, например:

```json
"berry": {
  "name": "Berry",
  "kind": "consumable",
  "stackable": true,
  "max_stack": 99,
  "icon_path": "items/consumables/berry.png",
  "description": "A wild berry."
}
```

Загрузка иконок идет через `get_item_icon(...)` из `catalog.py`.

Особенности:

- иконки кешируются;
- путь считается от каталога `assets/`;
- если файла нет, UI показывает текстовую заглушку.

## Валютные предметы

Предмет можно сделать "физической валютой", которая находится в мире как лут, но не занимает слот в инвентаре.

Для этого обычно используются поля:

- `kind: "currency"`
- `wallet_key`

Пример:

```json
"knowledge_shard": {
  "name": "Knowledge Shard",
  "kind": "currency",
  "stackable": true,
  "max_stack": 999999,
  "icon_path": "items/currency/knowledge_shard.png",
  "wallet_key": "knowledge_shards",
  "description": "A shard of knowledge used to unlock advanced recipes."
}
```

При подборе такой предмет не кладется в инвентарь, а сразу добавляется в соответствующий счетчик игрока.

## Важные правила

- `quest`-предметы автоматически стараются попадать в сюжетный инвентарь;
- обычные предметы не должны лежать в сюжетных слотах;
- рюкзак дает бонус к вместимости через `inventory_capacity_bonus`.

## Что менять здесь

- новый предмет;
- новый тип предметов;
- новые ограничения инвентаря;
- логика стекования и вместимости;
- подключение новых иконок предметов.
